# **Echo Minimal (闪记胶囊 Web 版) 完整布局 UI 设计说明**

## **1\. 空间布局概览 (The Layout Architecture)**

页面采用“非对称沉浸式”布局，将高频的“输入交互”置于视觉中心，将低频的“回顾功能”轻量化分布在四周。

* **中央区域 (Focus Zone)**：包含 **AI 开场白** 与 **动态对话框**。这是主要的交互阵地。  
* **右侧区域 (Stream Zone)**：**已记录的卡片流**。记录已定稿的想法，作为对话的沉淀。  
* **左侧区域 (Review Zone)**：**真实日历与今日总结**。提供时间维度和内容聚合的回溯。

## **2\. 核心元素与交互描述**

### **2.1 动态输入胶囊 (Dynamic Conversation Capsule)**

* **位置**：屏幕中心。  
* **设计**：“动态增长”的极简对话区。  
* **对话交互逻辑**：  
  * **第一回合**：AI 发起开场白，用户回复。  
  * **轻量衔接**：用户发送后，文字向上浮动，AI 随即在下方给出一句短促的反馈或追问（不超过 15 字），光标始终聚焦在下方。  
  * **视觉隔离**：对话过程中的“临时对话”字体颜色较浅（\#AAAAAA），而右侧“已定稿”的卡片颜色较深（\#1D1D1F）。  
  * **定稿机制**：当对话结束或用户点击“存入”图标（或按下 Cmd/Ctrl \+ Enter），这段对话会像胶囊一样飞入右侧卡片流，中心区域重置为清爽状态。

### **2.2 已记录的卡片 (Thought Cards)**

* **位置**：屏幕右侧占宽 30% 的区域。  
* **设计**：采用瀑布流。如果是对话形式，卡片会以“对话组合”的形式展示，折叠显示主要内容。  
* **交互**：新卡片从中心飞入顶部，旧卡片依次下移。

### **2.3 真实日历 (Calendar & Time Travel)**

* **位置**：屏幕左上角。  
* **设计**：  
  * **年月选择**：顶部设有微小的 \< 2026.01 \> 切换器，位于日历居中位置。  
  * **交互反馈**：  
    * **过去日期**：点击切换至历史卡片。  
    * **未来日期**：点击后屏幕中心出现：**“别急，未来还未来呢”**。

### **2.4 今日总结 (Manual Digest \- 纯手动触发)**

* **位置**：右侧，卡片流上方，只显示：“今天你记录了X条想法"，右侧为手动触发今日总结的按钮。  
* **交互机制**：  
  * **收尾仪式**：手动点击后，AI 提取今日所有“单次记录”和“对话片段”的关键词，获取返回结果。

### **2.5 今天对话封存 (Finished Today, 纯手动触发)**
  * **手动确认**：用户确认后封存。

## **3\. 交互过程说明 (Interaction Journey)**

1. **唤醒**：页面载入，AI 以温和的问候作为开场白。  
2. **交流**：用户自由地与 AI 进行多轮对话，AI 会适时给出简明或适度展开的反馈提问（字数不做严格限制，视用户交流意愿适度调整），持续对话直到用户主动结束。  
3. **主动结束**：用户随时可以点击“今天就到这吧”按钮，结束本次对话。该对话会被打包成一张**卡片**，飞入右侧卡片流。  
4. **误触未来**：点击未来日期（如下周），系统温和提示：**“别急，未来还未来呢”**。  
5. **落款**：睡觉前，用户可以点击“手动总结”，AI 会总结今天的对话要点，并将结果存入日历。

## **4\. 对话设计的防压策略 (Anti-Pressure Design)**

* **防止冗长**：AI 的回复设定为“短回复优先”。如果用户不说话，AI 不会二次追问。  
* **主动断句**：中心区域显示对话框，当用户点击“今天就到这吧”，或在语音中提到想要结束今日对话，超过则自动提示“要存入今日卡片吗？”，防止对话变成一种负担。  
* **输入框常驻**：无论在对话哪一阶段，输入框永远是极简的一条线，没有复杂的发送按钮。

## **5\. 针对 OKR 的优化点**

* **针对 KR3 (情绪负担)**：虽然支持对话，但 AI 的设定是“温柔的树洞”，它不纠缠，用户可以随时通过存入卡片来终止对话。  
* **针对 交互流畅**：对话的发生不需要切换页面，直接在当前的 Focus Zone 向上生长，视觉上非常丝滑。
